I made the following changes so that SE_GameOver and SE_GameOver2 play at the same time.

Changes made:

  1. Updated AudioManager.cs:
    • Added GameOver2 to the SEType enum. It has been appended at the end of the enum so that existing audio settings are not shifted out of order.
  2. Updated CombatManager.cs:
    • Modified the code so that both SEType.GameOver and SEType.GameOver2 are called at the point where the player's HP reaches 0 and the game-over process runs.
  3. Updated the AudioManager prefab:
    • Added a GameOver2 entry to the clips list in the AudioManager prefab in the project, and assigned the provided SE_GameOver2 (AudioClip) to it. UseReverb has also been enabled to match the settings of the original SE_GameOver.

How it works:

AudioManager has a mechanism that prevents duplicate playback when the same SEType is called within the same frame; however, since GameOver and GameOver2 are defined as different SEType values this time, calling both from CombatManager simultaneously causes them to play overlapping each other.

This allows both sound effects to play at the same time when a game over occurs.
