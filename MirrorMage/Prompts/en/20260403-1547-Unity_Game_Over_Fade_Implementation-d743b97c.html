The fade-in effect for the Game Over screen is implemented through the cooperation of two scripts: PlayerController.cs (death animation and scene transition) and GameOverController.cs (fade-in in the new scene).

This project uses Unity UI Toolkit (UITK), and the visual effect is achieved by manipulating properties of a black panel (VisualElement) used for fading via script.

The specific implementation flow is as follows.

1. Screen Darkening on Death (PlayerController.cs)

When the player's HP reaches 0, the DeathRoutine coroutine is executed.

  • Starting the fade-out: It searches for a UI element named fadeOverlay in the current scene (Main).
  • Expanding from the center: It animates fadeOverlay's height from 0% to 100% and its top (top edge position) from 50% to 0%, producing an effect where a black curtain spreads outward from the center of the screen.
  • Scene transition: After the screen is completely black, SceneManager.LoadScene("GameOver") is called.

2. Fade-In on the Game Over Screen (GameOverController.cs)

When the GameOver scene is loaded, this script handles the fade-in (the process of removing the black curtain).

UI Structure (GameOver.uxml / GameOver.uss)

  • A VisualElement named fadeOverlay is defined inside GameOver.uxml.
  • Via USS (stylesheet), this element is configured to "cover the entire screen in black" and be placed in the "foreground (absolute positioning)."

Fade Processing Logic (GameOverController.cs)

The FadeInRoutine coroutine is started inside OnEnable.

private IEnumerator FadeInRoutine()
{
    // 遷移を感じさせるため、一瞬（0.1秒）黒い状態を維持
    yield return new WaitForSecondsRealtime(0.1f);

    float duration = 1.0f; // 1秒かけてフェード
    float elapsed = 0f;

    while (elapsed < duration)
    {
        elapsed += Time.unscaledDeltaTime;
        float t = elapsed / duration;
        
        // イーズアウト（Ease-Out）処理: 1 - (1-t)^2
        // 最初は速く、後半はゆっくり透明になる
        float easeT = 1f - Mathf.Pow(1f - t, 2f);
        
        // 不透明度（opacity）を 1.0 から 0.0 へ減少させる
        _fadeOverlay.style.opacity = 1f - easeT;
        yield return null;
    }

    // 完全に透明になったら要素を非表示にする
    _fadeOverlay.style.opacity = 0f;
    _fadeOverlay.style.display = DisplayStyle.None;
}

Implementation Points

  • Easing: Instead of simple linear interpolation, the formula 1f - Mathf.Pow(1f - t, 2f) is used to achieve a smoother, more natural fade-in.
  • Unscaled Time: Because Time.unscaledDeltaTime is used, the animation plays without being affected by game pauses (Time.timeScale = 0) or similar.
  • Leveraging UITK: By directly manipulating style.opacity and style.display from C#, the entire implementation is self-contained within the UI system, with no need to write shaders.
