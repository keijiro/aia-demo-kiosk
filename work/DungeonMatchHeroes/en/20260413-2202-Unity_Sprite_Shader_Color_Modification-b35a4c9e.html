If so, is there any way to change a sprite to a color brighter than the texture color? For example, is it possible to make a character completely white?

🤖 Assistant

Yes, your understanding is correct. In Unity's standard sprite shaders (such as Universal Render Pipeline/2D/Sprite-Unlit-Default in URP), the Color property of SpriteRenderer is applied to the texture color via multiplication (Multiply).

Why can't you make it brighter?

With multiplication, the formula is: resulting color value = texture color × property color. Since each color component (R, G, B) is normally handled in the range of 0.0 to 1.0, multiplying by the maximum value of white (1, 1, 1) only gives you "the original texture color." For parts of the texture that are originally dark (e.g., 0.2), no matter what color you multiply by, you cannot reach 1.0 (pure white).

How to make a character completely white

To make a character completely white (such as a hit-flash effect), you need to use a shader that supports linear interpolation (Lerp) or Additive blending instead of multiplication.

The main approaches for achieving this in your current project are as follows:

  1. Create a custom shader with Shader Graph (recommended) Since you are using URP, creating a sprite shader in Shader Graph is the common approach.

    • Multiply the output of Sample Texture 2D by Vertex Color (the SpriteRenderer's color).
    • Connect the result to the A input of a Lerp node, and connect pure white to B.
    • Create a property called Flash Amount (0–1) and connect it to the T input of Lerp.
    • This way, setting Flash Amount to 1 will make the sprite pure white regardless of the original color.

  2. Change the material's Blend Mode (simple approach) If you only need to make the sprite white, you can prepare a material with its Blend Mode set to Additive and swap to it temporarily, which adds color to the existing pixels and causes them to blow out to white. However, this looks more like "making it glow" rather than "painting it over with white."

  3. Use the Sprite-Lit shader with HDR If you are using lighting (Sprite-Lit), you can set the material's color to HDR and raise the Intensity to blow it out to white, but since this is not easy to control from the standard SpriteRenderer properties, it is not a common approach.

In conclusion: Simply changing the Color property with the current standard shader cannot make a sprite brighter than its texture. If you want to make a character completely white as a visual effect, we recommend creating a custom shader in Shader Graph that includes Lerp (interpolation) processing.

When there are many sprites, as in a Match-3 game, there are several approaches considered "best for performance" that achieve the "flash to white" effect while maintaining batching.

The main reason batching is not broken is that vertex color (SpriteRenderer.color) is sent as vertex data.

  • Design idea: On the shader side, treat a specific value of the vertex color (e.g., Alpha) as "color intensity."
  • Logic: result = lerp(texture color * vertex color.rgb, vertex color.rgb, vertex color.a)
  • Merits:
    • Since you only change SpriteRenderer.color, batching is not broken at all.
    • No additional properties need to be sent to the GPU.
  • Demerits: If the sprite originally needs transparency (Alpha), Alpha can no longer be used for this purpose.
    • Workaround: Hacks such as setting one of the RGB values above 1.0 (HDR) are possible, but since the standard SpriteRenderer clamps values to 0.0–1.0, some ingenuity is required.

2. Use custom properties with GPU Instancing enabled

When creating a property in URP's Shader Graph, set that property's "Override Property Declaration" to "Instanced".

  • Design idea: Create a property such as _FlashAmount (0 to 1) and set it to Instanced.
  • Logic: Compute with lerp or add as in approach 1.
  • Merits:
    • With GPU Instancing, objects with different property values can be drawn in a single Draw Call as long as they share the same material (and the same texture).
  • Demerits: When using the URP 2D Renderer, you need to be careful about the priority and behavior of the SRP Batcher versus GPU Instancing, though it is more flexible than 3D-style batching.

3. Material swap (acceptable for short-duration effects)

For purposes such as "flash white for just 0.1 seconds upon receiving a hit," it can actually be the simplest and most practical choice to simply "accept that batching will break."

  • Approach: Prepare a dedicated all-white material in advance and swap spriteRenderer.sharedMaterial only at the moment of a hit.
  • Reasoning: Even if there are hundreds of sprites in a Match-3 game overall, only a few will be "flashing" at the same time. The increase of a few Draw Calls is a negligible impact on modern mobile and PC hardware.

4. Using a "white texture" from pre-rendering (extreme optimization)

If the target characters are limited, another option is to include a "white silhouette image of the character" in your Sprite Atlas and swap SpriteRenderer.sprite to that white image only when you want to flash.

Recommended approach

If you have a situation where "everyone is always glowing slightly in different colors," then creating "2. A Shader Graph shader with GPU Instancing support" is the straightforward solution.
