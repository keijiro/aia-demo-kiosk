이 프로젝트에는 공식 glTFast (com.unity.cloud.gltfast) 패키지가 도입되어 있으며, .glb는 GltfImporter를 통해 임포트되고 있습니다. 이에 대해 다음 절차로 접근할 수 있습니다.

기술적인 실현 방법과 접근 방식

1. 텍스처 추출 및 외부 에셋화

  1. 서브에셋 가져오기 AssetDatabase.LoadAllAssetsAtPath(glbPath)를 사용하여 .glb 파일에 내포된 Texture2D 오브젝트를 가져옵니다.
  2. 픽셀 데이터 출력 가져온 텍스처의 픽셀 데이터를 프로젝트 폴더(예: Assets/Models/Textures/)에 .png 또는 .tga 파일로 저장합니다.
    • ※ 텍스처가 직접 읽기 불가능한 설정(Readable이 아닌 경우 등)이더라도, 에디터 상에서 일시적으로 RenderTexture에 렌더링하여 읽어오는(Blit) 방식으로 안전하게 추출할 수 있습니다.
  3. 텍스처 압축 적용 추출한 .png 등을 AssetDatabase.ImportAsset하면 일반 텍스처 에셋으로 인식됩니다. 이를 통해 Unity 표준 TextureImporter를 거쳐 플랫폼별 텍스처 압축(ASTC, DXT, ETC2 등)을 적용하거나, 최대 해상도(Max Size)를 제한할 수 있게 됩니다.

2. 머티리얼 추출 및 교체

  1. 머티리얼 에셋 생성 마찬가지로 .glb 내에서 Material을 읽어와, 그 복제본(신규 머티리얼)을 AssetDatabase.CreateAsset으로 외부 에셋(.mat)으로 저장합니다.
  2. 텍스처 참조 교체 복제한 머티리얼에 대해, 1에서 추출한 외부 텍스처 에셋으로의 참조를 스크립트에서 자동으로 재할당합니다(예: material.SetTexture("_BaseMap", extTexture)).

3. 모델(GLB)에 머티리얼 재할당

추출한 머티리얼을 모델에 적용하는 방법으로 다음 중 하나를 채택합니다.

  • 임포터에서의 리매핑(권장): GltfImporter의 OnPostprocessMaterial 등의 임포트 파이프라인을 활용하거나, 임포트 설정의 External Objects(Material Map)를 스크립트에서 수정하여 GLB에 외부 머티리얼을 바인딩합니다.
  • 프리팹화를 통한 교체: 원본 GLB를 인스턴스화한 프리팹을 생성하고, 해당 Renderer 컴포넌트에 외부화한 머티리얼을 할당하도록 설정합니다.

이점 및 기대 효과

  • 빌드 크기·메모리의 대폭 절감: GLB 내부의 비압축 텍스처(특히 고해상도 PNG/JPG 등)를 GPU에 최적화된 압축 형식으로 변환할 수 있어, 빌드 크기 및 런타임 VRAM 사용량을 크게 줄일 수 있습니다.
  • 머티리얼 자유로운 커스터마이징: 머티리얼이 에셋으로 독립되므로, 셰이더 변경이나 프로퍼티 조정을 Unity 에디터 상에서 평소와 같이 자유롭게 할 수 있게 됩니다.

일반적으로 읽기 허용이 되지 않은 텍스처에 대해 GetPixels() 등을 호출하면 오류가 발생합니다. 이를 회피하기 위해, 임시 RenderTexture（렌더 텍스처） 를 생성하여 텍스처를 한 번 그곳에 렌더링한 후, 이를 CPU가 읽을 수 있는 Texture2D 로 복사하는 처리를 수행했습니다.

readableTex.ReadPixels(new Rect(0, 0, srcTex.width, srcTex.height), 0, 0);
readableTex.Apply();

// 4. 後片付け
RenderTexture.active = previous;
RenderTexture.ReleaseTemporary(rt);

// 5. PNG形式のバイト配列に変換
byte[] bytes = readableTex.EncodeToPNG();
Object.DestroyImmediate(readableTex);
